# 파스타 타이머 (뽀모올리)

**면을 고르면 타이머가 돌고, 기다리는 동안 그 파스타의 레시피를 보는 웹앱.**

<img src="docs/screenshots/3-running.png" width="300" alt="삶는 중 화면 — 진행 링 위를 뽀모가 돈다">

> **뽀모** 면부터 골라. 시간은 내가 잴게.
> **올리** …물 먼저 끓이고.

집에서 파스타 삶다가 "타이머 옆에 레시피도 있으면 편하겠는데" 싶어서 만들었어요.
방향은 직접 잡고 구현은 AI랑 바이브 코딩으로 했습니다. 아직 배포 전, 지금은 로컬에서만 돕니다.

## 뭐 하는 앱이냐면

1. 스파게티, 펜네, 푸실리 같은 면 10종 중에 하나 고르고
2. 알덴테나 기본 시간 중에 고르면 (봉지 시간이 다르면 직접 입력)
3. 면 넣는 순간 시작 — 타이머는 위에 붙어 있고, 아래로 내리면 소스 레시피

끝나기 1분 전에 미리 한 번, 다 되면 소리·진동·알림으로 또 한 번 알려줍니다.

> **뽀모** 복잡한 기능은 없어. 면만 안 퍼지면 성공이야.
> **올리** 동의.

## 몇 가지 챙긴 것

주방에서 쓰는 물건이라 이런 걸 신경 썼어요.

- 손에 물이 묻은 채로도 누르기 편하게 **버튼을 크게** (최소 48px)
- 조리대에서 좀 떨어져서 봐도 잘 보이게 **글씨도 크게** (본문 15px 아래로 안 내려감)
- 레시피 대부분이 면수를 쓰는데 알람 울린 뒤엔 이미 손이 바쁘니까 **1분 전에 미리 알림**
- 다른 앱 갔다 와도 시간이 안 어긋나게 (타임스탬프 기준으로 계산)
- 스파게티는 굵기(1.4~1.9mm)에 따라 삶는 시간이 최대 5분까지 차이 나서 **굵기별로 따로**

> **올리** 물은 면 100g에 1L. 소금도 잊지 말고.
> **뽀모** 그건 앱이 알려줘.

## 뽀모랑 올리

토마토랑 올리브. 파스타에 제일 많이 들어가는 둘을 캐릭터로 만들었고, 앱 이름도 여기서 나왔습니다.
크레파스로 아이가 그린 것 같은 그림체로 통일했어요. 삶는 동안엔 뽀모가 진행 링 위를 돌고, 다 되면 둘 다 폴짝 뜁니다.

## 로컬에서 돌리기

```bash
bun install
bun run dev      # localhost:3000
bun test         # 타이머 계산 테스트
bun run build    # 정적 빌드 → out/
```

## 만든 사람 메모

- Next.js 15 정적 export + PWA. 서버도 DB도 없어서 운영비가 안 듭니다.
- **아직 배포 전이고 실사용자 데이터도 없습니다.** 토스 미니앱 출시를 준비 중이에요.
- 삶는 시간은 시중 건면 표기 기준 기본값이라 브랜드마다 다를 수 있어요. 그래서 직접 입력을 넣어놨습니다.
- 광고나 제휴 링크는 안 붙였습니다. 주방에서 쓰는 물건에 광고 끼면 별로라서요.
