공식 [비게임 출시 체크리스트](https://developers-apps-in-toss.toss.im/checklist/app-nongame.html) 전 항목 대조.

### 보안 및 안정성 — 전부 통과
| 요건 | 상태 | 근거 |
|---|---|---|
| 외부 코드 실행 금지 | ✅ | `eval`·`new Function`·`dangerouslySetInnerHTML` 0건 |
| 히스토리 조작 금지 | ✅ | `pushState`·`replaceState`·`location.replace` 0건 |

| 요건 | 상태 | 근거 |
|---|---|---|
| 진입 시 바텀시트 자동 열림 금지 | ✅ | 바텀시트·모달 미사용 |

### 내비게이션 바 — 대응 완료, 실기기 확인 필요
| 요건 | 상태 | 비고 |
|---|---|---|
| 자체 뒤로가기와 토스 뒤로가기 중복 금지 | ✅ | `NEXT_PUBLIC_TARGET=toss` 빌드에서 자체 헤더 숨김 (검증: 토스 빌드 0건 / 웹 빌드 1건) |

| 브랜드 로고·미니앱 이름 중앙 표시 | ⏸ | 콘솔 등록 시 자동. 우리 화면은 파스타 이름을 타이머 카드가 흡수 |

## ⏸ 2단계 이후: 값을 받으면 Claude가 수행

```bash
bunx ait init --app-name <appName>     # granite.config.ts 생성
bunx ait token add --api-key <API키>    # 배포 토큰 등록
NEXT_PUBLIC_TARGET=toss bun run build  # 정적 빌드
bunx ait build                          # .ait 아티팩트
bunx ait deploy                         # 업로드
```

- [ ] `granite.config.ts` 작성 — appName·displayName·primaryColor(`#D95B43`)·icon·navigationBar 설정
- [ ] 샌드박스 앱 설치 → 갤럭시 실기기 테스트
- [ ] **샌드박스에서 반드시 확인할 3가지**
  - [ ] localStorage 유지 여부 (안 되면 타이머 복원 로직 대체 필요)
  - [ ] 하단 토글이 토스 탭바와 겹치거나 오인되지 않는지
  - [ ] 첫 화면 뒤로가기 시 미니앱이 정상 종료되는지
- [ ] 콘솔의 "앱 내 기능" 등록 + 스킴 연결 확인
- [ ] 심사 제출 → 결과 대응
