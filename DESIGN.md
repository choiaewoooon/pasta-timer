> 승인: 2026-07-16, 사용자 최재원 — 시안 "A 트라토리아 + C의 진행 링" 조합 확정.

> 모든 UI 작업 전에 이 문서를 먼저 읽는다. 여기서 벗어나려면 사용자 승인 필요.

## 제품 컨텍스트
- **무엇**: 파스타 종류별 삶기 타이머 + 레시피, 한국어 모바일 PWA
- **누구**: 집에서 파스타 해 먹는 요리 초보~중수, 주방에서 손에 물 묻은 채 사용
- **기억에 남길 한 가지**: "뽀모와 올리가 같이 파스타를 삶아주는 앱" —
  캐릭터 듀오가 곧 앱 이름(뽀모올리). 뽀모=뽀모도로(이탈리아어 토마토)의 애칭

## 미학 방향
- **방향**: 이탈리아 가정식 부엌의 온기 × Wabi식 절제된 구조 (Organic × Playful)
- **원칙**: **구조는 절제, 캐릭터는 발랄.** 레이아웃·컬러 사용은 미니멀하게 유지하고,
  성격 표현은 토마토·올리브 캐릭터에게만 맡긴다
- **장식 수위**: intentional — 크림 배경, 은은한 토마토 패턴은 빈 상태·완료 화면에만
- **레퍼런스**: `/Users/choejaewon/project/design-references/` → brand/tomato-olive-mood,
  app/wabi-meal-planner

## 타이포그래피
- **인사말·파스타 이름 (Display)**: 고운바탕 (Gowun Batang) — 세리프의 온기,
  Wabi의 "Hello there!" 공식
- **본문·버튼·라벨**: SUIT — 부드러운 한글 산세리프 (Pretendard 기본값 회피 결정)
- **타이머 숫자**: Fraunces — 세리프 숫자로 모든 타이머 앱과 차별화 (승인된 리스크).
  반드시 `font-variant-numeric: tabular-nums`
- **워드마크**: "뽀모올리" — SUIT 800, **투톤**: '뽀모'는 `--red`, '올리'는 `--sage`
  (캐릭터 듀오가 이름에 그대로 박힘). 라틴 보조 표기 "POMO & OLI" 대문자,
  letter-spacing 0.5em
- **명칭 표기 규칙** (2026-07-17):
  - 정식 명칭 = `파스타 타이머(뽀모올리)` — 스토어·콘솔·브라우저 제목·공유 미리보기
  - 축약 = `뽀모올리` — 홈 화면 아이콘 라벨(short_name), 앱 화면 안 워드마크
  - 화면 안에서는 축약형만 쓴다. 정식 명칭을 UI에 노출하면 브랜드가 흐려진다.
- 로딩: Google Fonts(Gowun Batang, Fraunces) + jsDelivr(SUIT Variable),
  `font-display: swap`, 서브셋 최적화
- 스케일: 타이머 64px / 화면 제목 22px / 카드 제목 17px / 본문 15px / 라벨·칩 12px
  (본문 15px 미만 금지 — 주방 가독성)

## 상태별 화면 규칙 (승인 조합의 핵심)
| 상태 | 표현 |
|---|---|

  `pomo.png`(기본) / `pomo-sleepy.png`(일시정지, 눈 감음) / `pomo-excited.png`(1분 전, 두 팔 듦).

### 앱 로고 (2026-07-19 확정)
**면이 그린 굵은 링 + 링 가운데 뽀모.** 원본 `assets/characters/logo-ring.png`.

- 요소는 둘인데 **파스타(면)·시간(링)·브랜드(뽀모)** 셋을 전달한다. 링은 앱의 시그니처
  진행 링과 같은 형태라 로고와 실제 화면이 같은 언어를 쓴다
- **올리는 로고에서 뺀다.** 40px에 캐릭터 둘은 면적이 안 나오고, 뽀모가 주인공이라는
  기존 배정에도 맞다. 올리는 대기·완료 화면과 레시피 마커에 계속 등장한다
- 선정 근거: 후보 4종(링 / 포크 / 듀오 / 기존 뽀모)을 **40·56·88·160px로 축소해 실측 비교**.
  Claude·Codex·Grok 3모델이 독립적으로 전부 링을 선택(Gemini는 헤드리스 이미지 읽기
  권한 문제로 미참여). 탈락 사유 — 포크: 손잡이가 가는 선이라 40px에서 소실 /
  듀오: 면 뭉치가 얼룩이 되고 캐릭터가 각각 작아짐 / 기존 뽀모: 파스타 신호 없음
- **로고는 큰 그림으로 고르지 않는다.** 실제 노출 크기(40~56px)로 줄여서 판단할 것
- **다크모드 로고는 만들지 않는다** (2026-07-19 실측). 배경을 브라운(#3D2C24·#2A1F1A)으로
  깔아보니 뽀모의 가는 팔다리에 들어간 진한 외곽선이 배경에 묻혀 **56px에서 팔다리가 사라진다.**
  크림 타일은 다크 UI에서 밝게 잘 읽히므로 그대로 쓴다. 제대로 하려면 캐릭터 외곽선을
  밝은 톤으로 다시 그려야 하는데 그건 v2 다크모드 재설계 범위다 (로고만 따로 손대면
  앱 안 캐릭터들과 어긋난다)

### 아이콘 세트 굽기
원본 하나에서 `docs/store-assets/tools/build-icons.py`로 전부 생성한다.
- 일반(192·512·apple-touch 180·favicon 32): 내용물 **80%**
- **maskable(192·512): 내용물 60%** — 안드로이드가 잘라내는 세이프존(안쪽 80% 원)에
  들어와야 링이 안 잘린다. 원형 로고라 여유가 있지만 규격은 지킨다

### 비의인화 오브젝트 층 (2026-07-19 신설)
캐릭터를 한 화면에 두 번 못 쓰는 대신, **얼굴 없는 크레파스 사물** 층을 따로 둔다.
파스타 면 아이콘 10종이 이미 쓰던 방식(`의인화 없이 면 모양만`)을 아이콘 전반으로 넓힌 것.

- `assets/ui-icons/` 원본 → `public/ui-icons/` 서빙(64px, **투명 배경**)
- `water-salt.png` 물·소금 안내 카드 / `tomato-plain.png` 뽀모의 팁·타이머 알림 배너 /
  `olive-branch.png` 레시피 섹션 마커(세이지=레시피 악센트 배정)
- **시스템 이모지 사용 금지.** OS마다 다르게 렌더돼 크레파스 화풍을 깬다.
  기존 `💧`·`🍅`(2곳)는 전부 이 층으로 교체 완료
- 표시 크기는 **28px 이상**. 20px 아래로 내리면 소금 알갱이·올리브 잎이 뭉개진다(실측)

### 에셋 생성·가공 파이프라인 (검증됨, 2026-07-19)
1. **생성 — Codex 내장 `image_gen`.** `cdx "..."`로 요청. 스타일 정본 이미지의 절대경로를
   프롬프트에 넣어 "먼저 열어 보고 같은 화풍으로"라고 지시해야 캐릭터가 유지된다
2. **배경은 마젠타 `#FF00FF` 크로마키로 요청.** image_gen은 투명 배경을 직접 못 만든다
3. Codex 세션은 읽기 전용이라 지정 경로 저장이 막힌다 → `~/.codex/generated_images/`에서 꺼내온다
4. **알파 추출**: `docs/store-assets/tools/dechroma.py` — 단색 매칭이 아니라 '마젠타 정도'를
   연속값으로 계산하고 스필(자주색 테두리)까지 제거한다
5. **프레이밍 정규화**: `tools/normalize.py` — 내용물이 프레임의 **0.70**을 차지하게 맞춘다.
   원형 크롭(`borderRadius:50%`)의 내접원에 들어오려면 대각선 기준 0.707 이하여야 한다.
   이걸 안 하면 팔 든 포즈가 잘리고 상태 전환 때 캐릭터 크기가 튄다

### 이미지 용량 규칙 (2026-07-19)
서빙 이미지는 **표시 크기의 2배**로 굽는다. 이전에 44px로 표시되는 `pomo.png`가 1254px·1.7MB였고
캐릭터 폴더만 3.6MB였다 → 281KB로 정리(-92%). 토스 심사의 '네트워크 급증 없음' 항목에 직결된다.
- **파스타 면 아이콘 10종**: `assets/pasta-icons/*.png` (원본 1024²+),
  `public/pasta-icons/*.png` (256px 서빙용) — 같은 크레파스 화풍, 밀색 #E8C97A 톤,
  의인화 없이 면 모양만 (2026-07-16 사용자 요청, Codex 생성). 홈 카드 왼쪽 56px
- 표정은 단순하게 (점 눈 + 미소 + 블러시). 유치함 방지 규칙: 캐릭터는 한 화면에
  1회만 등장, 크기는 타이머 UI의 보조 수준 유지, 말풍선 대사 금지 (상태 라벨로만)

## 간격·레이아웃
- 베이스 8px. 밀도 comfortable~spacious. **터치 타깃 최소 48px** (주방 UX)
- 모바일 단일 컬럼. 데스크톱은 max-width 480px 중앙 배치 (모바일 앱 프레임 유지)
- 라운드: 카드 24px / 버튼·칩 pill(999px) / 폰 화면 패딩 16px
- 화면 구조: 상단 고정(sticky) 타이머 카드 + 하단 레시피 스크롤 (PRD 확정 사항)
- **하단 플로팅 토글 바** (2026-07-16 사용자 요청): 상세 화면 하단에 fixed로 고정된
  타이머/레시피 pill 토글 — 스크롤과 무관하게 항상 표시, 현재 구간 자동 하이라이트.
  Wabi의 플로팅 pill 바텀 바 패턴 (레퍼런스: app/wabi-meal-planner)

## 모션
- 접근: intentional — 기능적 전환은 절제 (150~250ms, ease-out)
- 예외 규칙: **캐릭터와 진행 링에만 표현적 모션 허용** (링 진행, 캐릭터 idle 바운스,
  완료 점프 400~700ms)
- 타이머 숫자는 절대 애니메이션하지 않는다 (가독성 우선)

## 금지 목록 (AI 슬롭 방지)
- 순백 배경, 순검정 텍스트, 보라 그라데이션, 그라데이션 버튼
- 기계적 모노스페이스 타이머 숫자, 3컬럼 아이콘 그리드
- 화면당 2개 이상의 레드 액션, Inter/Roboto/Pretendard 기본값 회귀
