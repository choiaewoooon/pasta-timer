## 콘솔 MCP 연결됨 — 다만 등록을 자동으로 쏘면 안 된다 (2026-07-19 확인)

`apps-in-toss-console` MCP가 인증된 상태로 붙어 있어 워크스페이스·미니앱 조회와 등록이
API로 가능하다. **그런데 지금 바로 `miniapp_create`를 호출하면 안 된다.** 이유는 아래.

### 확인된 계정 상태
| 항목 | 값 |
|---|---|
| 워크스페이스 | `코블린` (id **57715**), 2026-07-07 생성 |
| 라이선스 | PERSONAL |
| **인증 상태** | `verified: false`, `reviewState: DRAFT` ← **미완료** |
| 등록된 미니앱 | 0개 |

### 🔴 차단 1 — 워크스페이스 인증이 DRAFT
워크스페이스가 아직 인증 전이다. MCP에는 인증 제출 도구가 없다(`workspace_update`는 이름 변경뿐).
**본인 명의 토스앱 인증이라 사용자만 할 수 있다.**

**사용자 진행 방법** ([콘솔](https://apps-in-toss.toss.im/) → **파트너 정보** 메뉴):
- 우리는 **사업자 등록이 필요 없다.** 공식 문서: *"사업자가 없는 개인 개발자인 경우에는
  사업자 등록을 하지 않아도 돼요."* 사업자가 필요한 6개 기능(토스 로그인·비즈월렛·프로모션·
  인앱광고·토스페이·인앱결제)을 이 앱은 하나도 안 쓴다 → 본인인증만으로 진행
- 사업자등록증을 내는 경우 발급까지 보통 3일. 우리는 해당 없음
- 인증 전에도 **테스트 앱 생성·개발은 가능**하다

> 미확인: 워크스페이스가 DRAFT인 상태에서 정식 미니앱 등록이 되는지는 공식 문서에 없다.
> 인증을 먼저 끝내고 등록하는 순서가 안전하다. 인증 과정에서 예상 밖 요구사항이 나오면
> 그 시점에 등록 방식(콘솔 웹 vs MCP)을 다시 정한다.

### 🔴 차단 2 — 카테고리 ID를 모른다
`miniapp_create`는 `categoryIds`/`subCategoryIds`에 **숫자 ID**를 요구하는데,
MCP에 카테고리 목록 조회 도구가 없고 공식 문서·커뮤니티에도 ID 표가 없다.
콘솔 웹 UI에서만 확인 가능한데 Claude의 브라우저 도구는 `toss.im`이 금융 도메인이라 차단된다.

### ⚠️ 위험 — 블라인드 호출이 위험한 이유
[커뮤니티 버그 제보(2026-07-09)](https://techchat-apps-in-toss.toss.im/t/mcp-bundle-upload-complete/4315),
운영자 "확인해볼게요"만 달리고 **수정 확인 없음**:

1. **`miniapp_create`는 검증 실패 응답을 주면서도 앱을 실제로 생성한다.**
   원문: *"에러 응답을 받았는데도 미니앱은 실제로 생성되어 있었습니다(miniAppId 발급, 목록 노출)."*
   → `appName`은 **영구 수정 불가**라서, 잘못 쏘면 `pomooli`를 망가진 레코드에 태워버린다.
2. 카테고리 설정이 MCP에서 실패하는 사례 보고 → **우회법은 콘솔 웹 UI 직접 입력**.
3. `bundle_upload_complete`가 항상 실패 → 번들 업로드도 콘솔 웹에서 해야 할 수 있다.

**결론: 등록(2단계)은 콘솔 웹 UI에서 사용자가 하는 편이 안전하다.** 값과 이미지는 전부
준비돼 있어 복사·업로드만 하면 된다 → [`store-assets/등록정보.md`](store-assets/등록정보.md).
등록 후 `appName`·`miniAppId`·API 키를 알려주면 3단계부터 Claude가 이어받는다.

## ⏸ 2단계 이후: 값을 받으면 Claude가 수행

```bash
bunx ait init --app-name <appName>     # granite.config.ts 생성
bunx ait token add --api-key <API키>    # 배포 토큰 등록
NEXT_PUBLIC_TARGET=toss bun run build  # 정적 빌드
bunx ait build                          # .ait 아티팩트
bunx ait deploy                         # 업로드
```

- [ ] `granite.config.ts` 작성 — appName·displayName·primaryColor(`#D95B43`)·icon·navigationBar 설정
- [ ] 샌드박스 앱 설치 → 갤럭시 실기기 테스트
- [ ] **샌드박스에서 반드시 확인할 3가지**
  - [ ] localStorage 유지 여부 (안 되면 타이머 복원 로직 대체 필요)
  - [ ] 하단 토글이 토스 탭바와 겹치거나 오인되지 않는지
  - [ ] 첫 화면 뒤로가기 시 미니앱이 정상 종료되는지
- [ ] 콘솔의 "앱 내 기능" 등록 + 스킴 연결 확인
- [ ] 심사 제출 → 결과 대응
